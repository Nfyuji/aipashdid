# PhishDetector — رؤية النظام وخريطة العمل الكاملة

## 1️⃣ هدف النظام

| الهدف | الحالة |
|-------|--------|
| كشف **رسائل البريد الاحتيالية** (Phishing, Spoofing) | ✅ مُنفَّذ (قواعد + كلمات مفتاحية + روابط + روابط مختصرة) |
| كشف **رسائل السبام** ذات الروابط/المرفقات المشبوهة | ✅ جزئي (روابط ونص؛ تحليل المرفقات **مخطوط**) |
| تقييم **درجة خطورة** كل رسالة (0–100) | ✅ مُنفَّذ |
| تصنيف موحّد: **Legitimate / Spam / Phishing** | ✅ مُنفَّذ |
| مساعدة المستخدمين وفرق الأمن على **اتخاذ القرار بسرعة** | ✅ لوحات تحكم + تفسير التصنيف (Explainable) |

---

## 2️⃣ المكونات الرئيسية للنظام

| القسم | الدور | الحالة |
|-------|--------|--------|
| **جمع الرسائل** | استقبال البريد من المستخدمين أو النظام | ✅ مُنفَّذ (نموذج إدخال: مرسل، موضوع، نص) — مخطوط: ربط IMAP/API بريد مؤسسي |
| **تحليل المحتوى** | تحليل النص، الروابط، والمرسل | ✅ جزئي (نص + روابط + روابط مختصرة + نطاقات مشبوهة) — مخطوط: مرفقات، NLP/Embeddings |
| **تصنيف الرسائل** | Legitimate / Spam / Phishing + درجة خطورة | ✅ مُنفَّذ |
| **لوحات التحكم** | عرض النتائج حسب الدور | ✅ مُنفَّذ (Admin, Security, Manager, Employee) |
| **إجراءات تلقائية** | حجب، تنبيهات، اقتراح قرارات | ✅ جزئي (إشعار أدمن عند عالي الخطورة، قائمة حظر يدوية) — مخطوط: حجب تلقائي، اقتراح إجراءات |

---

## 3️⃣ خريطة عمل النظام (من الاستقبال إلى العرض)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  1) استقبال البريد                                                          │
│  ─────────────────                                                          │
│  • المستخدم يلصق المرسل + الموضوع + النص  →  /employee/analyze (نموذج)      │
│  • [مخطوط] ربط صندوق بريد (IMAP) أو API لسحب الرسائل تلقائياً               │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  2) تحليل المحتوى (محرك الكشف)                                              │
│  ────────────────────────────────                                           │
│  • معالجة النص: كلمات مفتاحية مشبوهة (عربي/إنجليزي)     ← مُنفَّذ (قواعد)   │
│  • تحليل الروابط: عدد الروابط، روابط مختصرة، نطاقات    ← مُنفَّذ           │
│    تشبه جهات معروفة (بنك، دفع، تحقق)                                        │
│  • لغة استعجالية (عاجل، فوري، الآن، …)                 ← مُنفَّذ           │
│  • [مخطوط] NLP: TF-IDF / Embeddings، تحليل مرفقات، سمعة المرسل              │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  3) تصنيف الرسائل وتقييم الخطورة                                            │
│  ──────────────────────────────────                                        │
│  • درجة خطورة 0–100                          ← مُنفَّذ                      │
│  • تصنيف: Legitimate / Spam / Phishing       ← مُنفَّذ                      │
│  • تفسير مختصر (Explainable AI): أسباب التصنيف ← مُنفَّذ (حقل reasons)      │
│  • [مخطوط] نموذج ML/DL للتصنيف التلقائي                                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  4) حفظ النتائج وإجراءات تلقائية                                            │
│  ────────────────────────────────                                           │
│  • حفظ في DB: AnalyzedEmail (مرسل، موضوع، خطورة، تصنيف، أسباب، حالة)        │
│  • إشعار الأدمن عند رسالة عالية الخطورة / Phishing        ← مُنفَّذ         │
│  • [مخطوط] حجب تلقائي، إرسال تحذير بريد، اقتراح إجراءات                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  5) العرض على لوحات التحكم                                                  │
│  ─────────────────────────────                                              │
│  • Admin:   كل الرسائل، التحليلات، الإحصائيات، سجل التهديدات، إدارة النظام  │
│  • Security: الرسائل الخطرة، قائمة الحظر، خريطة التهديدات، صندوق المشبوه     │
│  • Manager: أداء الفريق، تقارير القسم، عدد التحليلات الخطرة                 │
│  • Employee: صندوق الوارد، نتيجة التحليل (تصنيف + تفسير)، سجلي               │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4️⃣ كيف يعمل الذكاء الاصطناعي في النظام (الحالي vs المخطوط)

| المكوّن | الحالي | المخطوط |
|---------|--------|---------|
| **معالجة النصوص (NLP)** | قواعد وكلمات مفتاحية (مثل verify, urgent, password، عربي وإنجليزي) | TF-IDF، Embeddings، نماذج لغة |
| **تحليل الروابط** | عدد الروابط، روابط مختصرة (bit.ly, tinyurl, …)، نطاقات تشبه بنك/دفع/تحقق | سمعة الرابط، فك اختصار والفحص، قائمة سوداء/بيضاء |
| **تحليل المرفقات** | — | نوع الملف (.exe, .zip مخاطر)، سكان فيروسات، استخراج نصوص |
| **تصنيف الرسائل** | قواعد → Legitimate / Spam / Phishing حسب النقاط | نموذج ML أو Deep Learning للتصنيف التلقائي |
| **تقييم الخطورة** | 0–100 من قواعد مركّبة | نموذج تنبؤ باحتمالية الاحتيال |
| **تفسير النتيجة (Explainable AI)** | حقل «أسباب»: كلمات مشبوهة، روابط، لغة استعجالية، إلخ | توسيع التفسير (أي جملة/رابط ساهم في القرار) |
| **اكتشاف أنماط جديدة** | — | تعلم أنماط هجمات جديدة، تنبيه عند شذوذ |

## 5️⃣ المخرجات حسب نوع المستخدم (محققة في النظام)

| نوع المستخدم | ماذا يرى |
|---------------|----------|
| **Admin** | كل الرسائل، التحليلات، الإحصائيات، سجل التهديدات، إدارة المستخدمين، الإعدادات، النسخ الاحتياطي، القوالب، الرسائل الداخلية، المهام، الإشعارات |
| **Security** | الرسائل الخطرة، قائمة الحظر، خريطة التهديدات، تفاصيل الهجمات، صندوق المشبوه (آخر 50) |
| **Manager** | أداء الفريق، عدد التحليلات وعدد عالي الخطورة لكل موظف، تقارير القسم |
| **Employee** | صندوق البريد (التحليلات)، نتيجة التحليل لكل رسالة (تصنيف + تفسير)، سجل التحليلات الخاصة به |

---

## 6️⃣ مزايا إضافية (حالية ومخططة)

| الميزة | الحالة |
|--------|--------|
| إشعارات عند تهديد عالي الخطورة | ✅ إشعار للأدمن عند تصنيف Phishing / عالي |
| قائمة حظر (بريد، نطاق، رابط) | ✅ إدارة يدوية من قسم الأمن/الأدمن |
| تفسير التصنيف (لماذا احتيالي/مزعج/عادي) | ✅ حقل `reasons` في كل تحليل |
| اكتشاف أنماط جديدة للهجمات | 🔲 مخطوط (نموذج شذوذ أو ML) |
| اقتراح إجراءات تلقائية (حجب، تحذير) | 🔲 مخطوط |
| تحليل جماعي بسرعة لكل الأقسام | 🔲 مخطوط (معالجة دفعية + تقارير جاهزة) |

## 7️⃣ خلاصة

**النظام الحالي يحقق:**

- **الهدف الأساسي**: كشف رسائل التصيد والسبام، تقييم الخطورة، تصنيف Legitimate/Spam/Phishing، ومساعدة المستخدمين والأمن على اتخاذ القرار عبر لوحات تحكم وتفسير النتائج.
- **المكونات الرئيسية**: جمع (نموذج)، تحليل (قواعد + روابط + روابط مختصرة)، تصنيف، لوحات تحكم، وإجراءات جزئية (إشعارات، قائمة حظر يدوية).
- **الذكاء الاصطناعي الحالي**: قواعد قابلة للتفسير (كلمات مفتاحية، روابط، لغة استعجالية، روابط مختصرة، نطاقات مشبوهة) مع خريطة واضحة لما هو **مخطوط** (NLP، ML/DL، المرفقات، السمعة، الحجب التلقائي، اكتشاف أنماط جديدة).

لتوسعة النظام لاحقاً: إضافة طبقة NLP/ML، ربط مصدر بريد (IMAP/API)، تحليل المرفقات، وحجب/تحذير تلقائي حسب السياسات.
